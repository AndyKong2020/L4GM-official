## 1. 技术栈梳理
- 主语言:Python / PyTorch。
- ML 框架:PyTorch + torch_npu; ImageDream 使用 diffusers/transformers 风格 pipeline; L4GM 主体为自定义 U-Net + Gaussian renderer。
- CUDA 依赖:原始路径依赖 CUDA/xFormers/gsplat;本次 NPU 适配后 xFormers 改为 PyTorch attention fallback,gsplat 改为功能 fallback。
- 自定义核(.cu / C++ 扩展):仓内推理主路径无必须编译的自定义 C++/CUDA 扩展;`gsplat` 为外部 CUDA rasterizer,当前 Ascend 环境不可用。
- 第三方库:diffusers、transformers、safetensors、torchvision、opencv、imageio、kiui、tyro。
- 模型权重 / 来源:`recon.safetensors`、`interp.safetensors`、ImageDream `image_encoder/text_encoder/unet/vae` 本地权重。权重不纳入 git,部署时放在 `pretrained/`。
- 仓内 benchmark:无独立 `benchmark/`、`eval.py` 或标准分数脚本。训练/验证 PSNR 依赖外部 Objaverse 渲染数据和 datalist,本轮未复现。

## 2. 部署步骤
- [x] 依赖安装:使用挂载盘虚拟环境,确认 `torch_npu` 可用。
- [x] 编译 / 构建:无本地扩展编译;通过 `python -m py_compile` 检查 NPU 适配文件。
- [x] 权重获取:所有 L4GM 与 ImageDream 权重放在 `pretrained/`,不写入系统盘。
- [x] NPU 适配改动(device、torch_npu、禁用 CUDA 核等):
  - `core/device.py`:统一 NPU 设备选择、cache 目录、autocast、empty cache、CLIP tensor materialize。
  - `infer_3d.py` / `infer_4d.py` / `main.py`:替换 CUDA-only device/autocast/cache,推理阶段禁用 LPIPS 权重下载。
  - `mvdream/pipeline_mvdream.py`:ImageDream `text_encoder`、`image_encoder` 迁移 NPU 前先 CPU materialize,绕过 checkpoint-backed tensor 直接 `.to(npu)` 卡死。
  - `mvdream/mv_unet.py`:xFormers 不可用时走 PyTorch attention fallback。
  - `core/gs.py`:CUDA `gsplat` 不可用时使用简化 NPU fallback rasterization。
  - `scripts/npu_env.sh` / `scripts/npu_smoke.py`:提供可重复 NPU 环境与 smoke。
- 命令:
```bash
source scripts/npu_env.sh
python scripts/npu_smoke.py

python infer_3d.py big \
  --workspace results_npu_materialized \
  --resume pretrained/recon.safetensors \
  --num_frames 1 \
  --test_path data_test/otter-on-surfboard_fg.mp4

python infer_4d.py big \
  --workspace results_npu_materialized \
  --resume pretrained/recon.safetensors \
  --interpresume pretrained/interp.safetensors \
  --num_frames 4 \
  --test_path data_test/otter-on-surfboard_fg.mp4
```

## 4. NPU 亲和性
口径:Ascend 950PR、3510 架构、FP16/autocast、单进程单卡推理;后四张卡可见但未做多卡并行。950PR FP16 Cube 平衡点约 270 FLOP/Byte,Vector FP16 平衡点约 34 FLOP/Byte。平衡点仅用于初判,不替代上板 profiler。

| 指标 | 数值 |
|---|---|
| 能否在 NPU 跑通 | 3D 推理跑通;4D `num_frames=4/8/10` 跑通;`num_frames=11/12/16` 未跑通;当前脚本完整视频实测上限为 10 帧 |
| NPU 利用率 (npu-smi) | `num_frames=8` 采样:NPU Util 99-100%,AICore 95-100%,AIVector 65-84%,AICube 14-27%,HBM Usage 54%;另对实际 `forward_gaussians` 和 renderer fallback 分别采集 Level1+PipeUtilization profiler |
| HBM 占用 | 3D peak alloc/reserved 7.38/9.52 GiB;4D 4/8/10 帧为 14.79/17.17、45.97/56.38、68.96/92.49 GiB;11/12 帧 OOM peak alloc/reserved 为 79.87/80.17、58.16/89.65 GiB;16 帧 softmax 单次申请 64 GiB |
| 关键算子是否回退 CPU | 未观察到主干 CPU fallback;CLIP 权重迁移需 CPU materialize;CUDA `gsplat` 被 NPU fallback 替代 |
| 性能(吞吐/时延) | ImageDream 30-step 约 17.2 it/s;3D wall-clock 8.94s,内部生成 2.82s;4D 4/8/10 帧完整 150 帧 loop 为 64.86/75.29/78.83s |

**各计算单元逐条判定**:

| 单元 | 压力 | 判定 | 证据 |
|---|---|---|---|
| 算力(Cube,矩阵卷积) | Conv2d、Linear、attention matmul 为主干 | 中等亲和 | 3D 与 4D 10 帧可跑;Level1 profiler 中 BatchMatMul 12.2%、Conv2D 合计约 9.2%,AICube 采样 14-27%,说明整网不是纯 Cube 饱和 |
| 向量(Vector,归一激活) | GroupNorm、LayerNorm、SiLU、softplus、sigmoid、tanh、softmax、PSNR reduce | 条件亲和 | 8 帧采样 AIVector 65-84%;Level1 profiler 中 Cast 43.4%、Softmax 25.4%、GroupNormSilu 2.8%、Swish 2.1%,Vector/格式转换压力高 |
| 搬运(MTE/FixPipe) | tensor `.to(npu)`、cat/stack/repeat、contiguous、attention matrix materialize | 中到低 | Level1 profiler 加权 aic_mte2 约 15.1%,aiv_mte2/mte3 约 76.7%/71.6%;10 帧 reserved 已到 92.49 GiB,11+ 帧 HBM 峰值不可承受 |
| 通信(communication) | 单进程单卡 | 不适用 | 仓内无多卡通信路径;轻量 profiler step trace 通信时间为 0,未把 HCCL/多卡 allreduce 当成本项目验收项 |
| 调度(host/head) | ImageDream 30-step diffusion、4D 逐段循环、renderer Python loop | 中 | 4D 4/8/10 帧完整 loop 通过;8 帧采样计算段 NPU Util 接近满载;正式 profiler 覆盖 `forward_gaussians` 和单视角 renderer,端到端 wall-clock 覆盖视频编码与 Python 循环 |

**算子级摘要**:

| 算子/路径 | NPU 状态 | 说明 |
|---|---|---|
| `Conv2d`、`nn.Linear`、`GroupNorm`、`LayerNorm`、`F.interpolate` | 支持 | 覆盖 LGM 主干、ImageDream VAE/UNet、3D/4D 降帧成功链路 |
| `q @ k.T`、`attn @ v` | 条件支持 | matmul 可跑;显式 attention matrix 在 10 帧可跑,11/12/16 帧造成 HBM 峰值/OOM |
| `softmax(dim=-1)` | 条件支持 | 小规模可跑;11/12/16 帧 temporal attention 在此处 OOM |
| `Tensor.to("npu")` | 条件支持 | 普通 tensor 支持;CLIP checkpoint-backed weight 直接迁移超时 |
| `detach().clone().contiguous()` materialize | 支持 | 作为加载期规避方案,使 CLIP encoder 迁移 NPU 通过 |
| `scatter_add_`、mask、boolean indexing | 功能支持,不规则低亲和 | PyTorch 层 200k 点 scatter/mask microbench 通过;renderer Level1 profiler 中 NonZero/Gather/ScatterAdd/ScatterElements 均有 NPU kernel,但属于 Vector/不规则路径而非 Cube 主导 |
| `grid_sample`、PSNR reduce | 支持 | 补充算子用例通过,覆盖训练/评估中的视角采样和指标 reduce |
| LPIPS/VGG | 条件支持 | VGG features 与 LPIPS(`pretrained=False`)算子链路通过;预训练权重加载和质量对齐未测 |
| xFormers `memory_efficient_attention` | 不可用 | CUDA 扩展不可用,已走 PyTorch fallback |
| CUDA `gsplat.rendering.rasterization` | 不可用 | 以简化 fallback 替代,不等价官方视觉质量 |

**NPU 特有项**:
- tile/double buffer:Level1 profiler 已确认 forward 主干不是纯 Cube 饱和;10 帧接近可用 HBM 上限,11+ 帧 attention 的 `N^2` matrix 不可驻留,不能靠扩大融合解决。
- 对齐/小包:950PR 按 512B L2 cacheline + 4×128B sector 口径;未套用 A2 的 512B GM 经验曲线。
- layout 折叠:`view/reshape/permute` 不重复计 GM bytes;`contiguous/cat/repeat` 才按真实 materialization 计。
- 不规则负载:renderer fallback 的 scatter/mask/alpha blend 不用 Cube 平衡点判断;正式 profiler 中 renderer top op 为 Slice 10.7%、NonZero 8.9%、Mul 8.9%、Pack 7.1%、Cast 7.0%、GatherV2 5.5%、ScatterAdd 2.9%、ScatterElements 2.7%,亲和低于规整 Conv/GEMM。

- 算子回退清单:
  - xFormers CUDA attention 不可用 → PyTorch attention fallback。
  - CUDA `gsplat` 不可用 → 简化 NPU fallback rasterization。
  - CLIP checkpoint-backed tensor 直接迁移阻塞 → CPU materialize 后迁移。
- profiler 摘要:按注入采集路径执行正式 profiler。仓内无 `enable_profiler` 内置路径;采集配置为 `torch_npu.profiler._ExperimentalConfig(Level1 + PipeUtilization)`,profile 外预热,schedule `wait=0,warmup=0,active=3,repeat=1`,循环后额外 `prof.step()` 收尾。`forward_gaussians` profiler 验收结果:`kernel_details.csv` 3906 行/49 列,`trace_view.json` 合法,`op_statistic.csv`、`api_statistic.csv`、`operator_details.csv`、`step_trace_time.csv` 均生成;3 个 step computing 分别约 292.9/293.0/293.1 ms,communication 为 0;主要耗时:Cast 43.4%,Softmax 25.4%,BatchMatMul 12.2%,Conv2D 合计约 9.2%,GroupNormSilu 2.8%,Swish 2.1%,Transpose 1.6%。renderer profiler 验收结果:`kernel_details.csv` 222 行/49 列,统计 CSV 和 trace 均生成;3 个 step computing 约 270.9/268.8/266.1 us,communication 为 0;主要耗时:Slice 10.7%,NonZero 8.9%,Mul 8.9%,Pack 7.1%,Cast 7.0%,ClipByValueV2 6.9%,RealDiv 6.6%,GatherV2 5.5%,ScatterAdd/ScatterElements 约 5.6%。

## 5. 阻塞项
| 阻塞点 | 原因 | 是否硬阻塞 | CANN/AscendC 替代方案 | 兜底 |
|---|---|---|---|---|
| ImageDream CLIP encoder `.to(npu)` 卡死 | safetensors/diffusers loader 产生的 checkpoint-backed CPU tensor 与 torch_npu copy path 不兼容 | 否,已规避 | 加载后 materialize 为普通 contiguous tensor;或在 loader 层禁用 mmap/backing | 保持当前 `materialize_cpu_module_tensors()` |
| 4D 高帧数 OOM | PyTorch attention fallback 显式构造 `[N,N]` attention matrix;10 帧可完整跑通,11/12/16 帧在 temporal attention softmax OOM | 是,对 11 帧以上和 README 默认 16 帧配置 | online softmax、chunk attention、NPU SDPA/AscendC memory-efficient attention | 当前脚本以 `num_frames<=10` 做 NPU 功能复现 |
| CUDA `gsplat` 不可用 | 原 Gaussian rasterizer 依赖 CUDA 扩展 | 是,对官方视觉质量 | 实现 NPU tile binning + sort + alpha blend;评估 SIMD/SIMT mixed | 当前简化 scatter fallback 仅保证功能产物 |
| 仓内 benchmark 缺失 | 无官方 benchmark 脚本;PSNR 依赖外部 Objaverse 数据 | 否,但影响定量结论 | 补齐数据后跑 `main.py` eval 并 profile | 以功能验证和阻塞定位作为当前验收 |
| 训练 LPIPS 预训练权重与完整 eval 属外部依赖 | 已验证 grid_sample、PSNR reduce、VGG/LPIPS(no pretrained)算子链路;LPIPS 预训练权重需额外下载,Objaverse datalist/数据集未纳入当前复现 | 否 | 将权重和数据集放在挂载盘后跑 `main.py` eval + profiler | 当前只声明算子链路可用,不声明训练/评估指标完整复现 |

## 6. 结论
- 运行方案(NPU / NPU+CPU / CPU):NPU 为主。3D 推理和 4/8/10 帧 4D 推理已在 NPU 跑通;10 帧 4D peak reserved 92.49 GiB,11 帧开始触发 attention OOM;加载期 CLIP materialize 使用 CPU tensor 预处理;renderer 使用 NPU 功能 fallback。
- 亲和性结论:基础 Conv/Linear/Norm/interpolate/attention matmul、grid_sample、PSNR reduce、VGG/LPIPS(no pretrained)算子链路可用;高帧数 4D 的主阻塞是显式 attention matrix 和 softmax workspace 导致的 HBM 峰值,不是基础算子缺失。
- 当前可交付状态:可作为 NPU 功能复现报告;当前脚本完整视频实测上限为 10 帧;不能声明 README 默认 16 帧 4D、LPIPS 预训练权重质量或 CUDA `gsplat` 视觉质量等价已完成。
- 待办 / 风险:
  - 补 memory-efficient / chunk attention,优先验证 11/12/16 帧。
  - 替换简化 renderer,建立 NPU Gaussian rasterizer 与 CUDA 参考的质量对齐。
  - 进入性能优化阶段时,扩展 profiler 到 ImageDream diffusion 和完整视频端到端 trace。
  - 若需要指标,补 Objaverse 数据后跑 PSNR eval。
