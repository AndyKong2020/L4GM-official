## 1. 技术栈梳理
- 主语言:Python / PyTorch。
- ML 框架:PyTorch + torch_npu; ImageDream 使用 diffusers/transformers 风格 pipeline; L4GM 主体为自定义 U-Net + Gaussian renderer。
- CUDA 依赖:原始路径依赖 CUDA/xFormers/gsplat;本次 NPU 适配后 xFormers 改为 PyTorch attention fallback,gsplat 改为功能 fallback。
- 自定义核(.cu / C++ 扩展):仓内推理主路径无必须编译的自定义 C++/CUDA 扩展;`gsplat` 为外部 CUDA rasterizer,当前 Ascend 环境不可用。
- 第三方库:diffusers、transformers、safetensors、torchvision、opencv、imageio、kiui、tyro。
- 模型权重 / 来源:`recon.safetensors`、`interp.safetensors`、ImageDream `image_encoder/text_encoder/unet/vae` 本地权重。权重不纳入 git,部署时放在 `pretrained/`。
- 仓内 benchmark:无独立 `benchmark/`、`eval.py` 或标准分数脚本。训练/验证 PSNR 依赖外部 Objaverse 渲染数据和 datalist,本轮未复现。

## 2. 部署步骤
- [x] 依赖安装:使用挂载盘虚拟环境,确认 `torch_npu` 可用。
- [x] 编译 / 构建:无本地扩展编译;通过 `python -m py_compile` 检查 NPU 适配文件。
- [x] 权重获取:所有 L4GM 与 ImageDream 权重放在 `pretrained/`,不写入系统盘。
- [x] NPU 适配改动(device、torch_npu、禁用 CUDA 核等):
  - `core/device.py`:统一 NPU 设备选择、cache 目录、autocast、empty cache、CLIP tensor materialize。
  - `infer_3d.py` / `infer_4d.py` / `main.py`:替换 CUDA-only device/autocast/cache,推理阶段禁用 LPIPS 权重下载。
  - `mvdream/pipeline_mvdream.py`:ImageDream `text_encoder`、`image_encoder` 迁移 NPU 前先 CPU materialize,绕过 checkpoint-backed tensor 直接 `.to(npu)` 卡死。
  - `mvdream/mv_unet.py`:xFormers 不可用时走 PyTorch attention fallback。
  - `core/gs.py`:CUDA `gsplat` 不可用时使用简化 NPU fallback rasterization。
  - `scripts/npu_env.sh` / `scripts/npu_smoke.py`:提供可重复 NPU 环境与 smoke。
- 命令:
```bash
source scripts/npu_env.sh
python scripts/npu_smoke.py

python infer_3d.py big \
  --workspace results_npu_materialized \
  --resume pretrained/recon.safetensors \
  --num_frames 1 \
  --test_path data_test/otter-on-surfboard_fg.mp4

python infer_4d.py big \
  --workspace results_npu_materialized \
  --resume pretrained/recon.safetensors \
  --interpresume pretrained/interp.safetensors \
  --num_frames 4 \
  --test_path data_test/otter-on-surfboard_fg.mp4
```

## 3. 验证用例
- 输入数据:`data_test/otter-on-surfboard_fg.mp4`。
- 运行命令:见第 2 节;另用 `infer_4d.py --num_frames 16` 验证 README 默认 4D 配置的阻塞点。
- 期望输出:
  - 3D:原图、多视图 PNG、4 个渲染 PNG、3D MP4。
  - 4D:无插值/插值视频以及 fixed-view 视频。
- 实测输出:
  - `scripts/npu_smoke.py`:可见 4 张 NPU,`gaussians_shape=(1, 4096, 14)`,`gaussians_finite=True`。
  - recon/interp checkpoint:加载 `missing=0 unexpected=0`,reduced forward 输出 finite。
  - ImageDream:1-step 前向输出 `(5, 256, 256, 3)`,finite;30-step 3D 入口约 17 it/s。
  - `infer_3d.py big --num_frames 1`:通过,生成 3D PNG/MP4;wall-clock 8.94s,内部 `Generate 3D` 2.82s,HBM peak alloc/reserved 7.38/9.52 GiB。
  - `infer_4d.py big --num_frames 4`:通过,完成 150 帧 loop,生成 `wo_interp_*`、`w_interp_*` 及 fixed 视频;wall-clock 64.86s,HBM peak alloc/reserved 14.79/17.17 GiB。
  - `infer_4d.py big --num_frames 16`:未通过,temporal attention softmax 申请 64 GiB HBM 后 OOM。
- 与 CPU/GPU 基准对比(误差/一致性):未做逐像素 CPU/GPU 数值对比。当前验证为功能跑通、输出文件存在和关键 tensor finite;renderer fallback 不声明与 CUDA `gsplat` 视觉质量等价。

## 4. NPU 亲和性
口径:Ascend 950PR、3510 架构、FP16/autocast、单进程单卡推理;后四张卡可见但未做多卡并行。950PR FP16 Cube 平衡点约 270 FLOP/Byte,Vector FP16 平衡点约 34 FLOP/Byte。平衡点仅用于初判,不替代上板 profiler。

| 指标 | 数值 |
|---|---|
| 能否在 NPU 跑通 | 3D 推理跑通;4D `num_frames=4` 跑通;README 默认 4D `num_frames=16` 未跑通 |
| NPU 利用率 (npu-smi) | 未采集完整 profile;补测采集 torch_npu 峰值内存,功能验证后无本任务残留进程 |
| HBM 占用 | 3D peak alloc/reserved 7.38/9.52 GiB;4D `num_frames=4` peak alloc/reserved 14.79/17.17 GiB;16 帧 OOM 前 PyTorch active 约 36.93 GiB,单次 softmax 申请 64 GiB |
| 关键算子是否回退 CPU | 未观察到主干 CPU fallback;CLIP 权重迁移需 CPU materialize;CUDA `gsplat` 被 NPU fallback 替代 |
| 性能(吞吐/时延) | ImageDream 30-step 约 17.2 it/s;3D wall-clock 8.94s,内部生成 2.82s;4D `num_frames=4` 完整 150 帧 loop 64.86s,逐段 forward 约 0.01-0.08s |

**各计算单元逐条判定**:

| 单元 | 压力 | 判定 | 证据 |
|---|---|---|---|
| 算力(Cube,矩阵卷积) | Conv2d、Linear、attention matmul 为主干 | 中高亲和 | LGM smoke、3D、4D 降帧均通过;Conv/Linear 初判高于 950PR FP16 Cube 平衡点,但 tile 驻留待 profile |
| 向量(Vector,归一激活) | GroupNorm、LayerNorm、SiLU、softplus、sigmoid、tanh、softmax | 条件亲和 | 3D/4D 降帧通过;16 帧阻塞在显式 attention softmax workspace,不是基础 Vector 算子不可用 |
| 搬运(MTE/FixPipe) | tensor `.to(npu)`、cat/stack/repeat、contiguous、attention matrix materialize | 中到低 | 普通 tensor 迁移可用;checkpoint-backed CLIP tensor 直接迁移超时,materialize 后可用;16 帧 attention HBM 峰值过高 |
| 通信(communication) | 单进程单卡 | 无结论 | 未做 HCCL/CCU/URMA/多卡切分;通信路径待测 |
| 调度(host/head) | ImageDream 30-step diffusion、4D 逐段循环、renderer Python loop | 中 | 功能通过,但多 step launch/head 占比未 profile;图模式/npugraph 未测 |

**算子级摘要**:

| 算子/路径 | NPU 状态 | 说明 |
|---|---|---|
| `Conv2d`、`nn.Linear`、`GroupNorm`、`LayerNorm`、`F.interpolate` | 支持 | 覆盖 LGM 主干、ImageDream VAE/UNet、3D/4D 降帧成功链路 |
| `q @ k.T`、`attn @ v` | 条件支持 | matmul 可跑;显式 attention matrix 在 16 帧下造成 HBM 峰值 |
| `softmax(dim=-1)` | 条件支持 | 小规模可跑;16 帧 temporal attention 在此处 OOM |
| `Tensor.to("npu")` | 条件支持 | 普通 tensor 支持;CLIP checkpoint-backed weight 直接迁移超时 |
| `detach().clone().contiguous()` materialize | 支持 | 作为加载期规避方案,使 CLIP encoder 迁移 NPU 通过 |
| `scatter_add_`、mask、boolean indexing | 功能支持,亲和待测 | renderer fallback 可出图;属于不规则负载,950 SIMD/SIMT mixed 效率需 microbench |
| `grid_sample`、LPIPS/VGG、PSNR reduce | 未验证 | 训练/评估路径未跑 |
| xFormers `memory_efficient_attention` | 不可用 | CUDA 扩展不可用,已走 PyTorch fallback |
| CUDA `gsplat.rendering.rasterization` | 不可用 | 以简化 fallback 替代,不等价官方视觉质量 |

**NPU 特有项**:
- tile/double buffer:Conv/Linear 主干待 profiler 验证 L0/L1/UB 驻留;16 帧 attention 的 `N^2` matrix 不可驻留,不能靠扩大融合解决。
- 对齐/小包:950PR 按 512B L2 cacheline + 4×128B sector 口径;未套用 A2 的 512B GM 经验曲线。
- layout 折叠:`view/reshape/permute` 不重复计 GM bytes;`contiguous/cat/repeat` 才按真实 materialization 计。
- 不规则负载:renderer fallback 的 scatter/mask/alpha blend 不用 Cube 平衡点判断,需单独 microbench。

- 算子回退清单:
  - xFormers CUDA attention 不可用 → PyTorch attention fallback。
  - CUDA `gsplat` 不可用 → 简化 NPU fallback rasterization。
  - CLIP checkpoint-backed tensor 直接迁移阻塞 → CPU materialize 后迁移。
- profiler 摘要:未采集完整 profiler。当前结论基于功能运行、torch_npu 峰值内存、OOM 栈、组件拆分和 roofline 粗估;Cube/Vector/MTE/head 占比均列为待测。

## 5. 阻塞项
| 阻塞点 | 原因 | 是否硬阻塞 | CANN/AscendC 替代方案 | 兜底 |
|---|---|---|---|---|
| ImageDream CLIP encoder `.to(npu)` 卡死 | safetensors/diffusers loader 产生的 checkpoint-backed CPU tensor 与 torch_npu copy path 不兼容 | 否,已规避 | 加载后 materialize 为普通 contiguous tensor;或在 loader 层禁用 mmap/backing | 保持当前 `materialize_cpu_module_tensors()` |
| 4D README 默认 `num_frames=16` OOM | PyTorch attention fallback 显式构造 `[N,N]` attention matrix,softmax workspace 申请 64 GiB | 是,对默认 16 帧配置 | online softmax、chunk attention、NPU SDPA/AscendC memory-efficient attention | 使用 `num_frames=4` 做 NPU smoke/功能复现 |
| CUDA `gsplat` 不可用 | 原 Gaussian rasterizer 依赖 CUDA 扩展 | 是,对官方视觉质量 | 实现 NPU tile binning + sort + alpha blend;评估 SIMD/SIMT mixed | 当前简化 scatter fallback 仅保证功能产物 |
| 仓内 benchmark 缺失 | 无官方 benchmark 脚本;PSNR 依赖外部 Objaverse 数据 | 否,但影响定量结论 | 补齐数据后跑 `main.py` eval 并 profile | 以功能验证和阻塞定位作为当前验收 |
| 训练 LPIPS/grid_sample 未验证 | 推理禁用 LPIPS,训练增强未进入 | 否 | 单独验证 VGG/LPIPS、grid_sample、reduce/state layout | 仅声明推理亲和性 |

## 6. 结论
- 运行方案(NPU / NPU+CPU / CPU):NPU 为主。3D 推理和 4 帧 4D 推理已在 NPU 跑通;4 帧 4D 补测峰值 reserved 17.17 GiB,仍低于 16 帧 softmax OOM 阻塞;加载期 CLIP materialize 使用 CPU tensor 预处理;renderer 使用 NPU 功能 fallback。
- 亲和性结论:基础 Conv/Linear/Norm/interpolate/attention matmul 亲和;默认 16 帧 4D 的主阻塞是 attention 显式矩阵导致的 HBM 峰值,不是基础算子缺失。
- 当前可交付状态:可作为 NPU 功能复现报告;不能声明 README 默认 16 帧 4D 或 CUDA `gsplat` 视觉质量等价已完成。
- 待办 / 风险:
  - 补 memory-efficient / chunk attention,逐步验证 4/8/12/16 帧。
  - 替换简化 renderer,建立 NPU Gaussian rasterizer 与 CUDA 参考的质量对齐。
  - 补 profiler,拆 Cube/Vector/MTE/head 占比。
  - 若需要指标,补 Objaverse 数据后跑 PSNR eval。
